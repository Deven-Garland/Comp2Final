## Claude (claude.ai) Sessions

---

### Conversation 17
**Tool used:** Claude

**What we asked:** Needed all the core data structure files built from scratch — hash_table.py, linked_list.py, circular_buffer.py, bloom_filter.py, sorting.py, node.py, accounts.py, chat.py, and the other platform server files. Also needed a test file.

**What it gave us:** Full implementations of everything. HashTable uses FNV-1a hashing with separate chaining through LinkedList. BloomFilter uses an ArrayList bit array. sorting.py has both merge sort and insertion sort with key/reverse support. accounts.py chains HashTable and BloomFilter together. chat.py maps game sessions to CircularBuffers. Test file matches the lab PASS/FAIL style with a check() helper.

**What we actually used:** Kept all of it. Had to correct the author name in the headers mid-session — it put a wrong name in at first — then regenerated everything with Author: Ellie Lutz.

---

### Conversation 18
**Tool used:** Claude

**What we asked:** A bunch of bugs in Ellie's game at once: chat wasn't syncing between players, ghost players were sticking around after leaving, pressing SPACE in chat was triggering the attack, inventory was getting cut off, and sprites weren't loading.

**What it gave us:** Found the root cause for all five. Chat was broken because _session_id was never being set on the connection object so every message went out with game_id: None. Ghost players were happening because leave never actually closed the TCP socket to the C++ server. SPACE conflict needed an input_enabled flag on Character since the game uses key polling not event handling. Inventory was using hardcoded screen dimensions instead of reading from the surface. Sprite paths were relative and broke depending on which directory you launched from — fixed with GAME_DIR absolute paths using os.path.abspath.

**What we actually used:** Kept all five fixes. The GAME_DIR pattern is now used everywhere in the game folder.
