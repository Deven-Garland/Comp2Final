Conversation 7
Tool used: Cursor (Composer)

What I asked: Material for a networking presentation; why environment variables are still needed when SSH port-forwarding is already set up; teammates (e.g. MobaXterm) failing to connect when the client hard-coded the platform port.

What it gave me: Concept split: forwarding only creates local listening sockets; the app must still target the correct host:port pair. Two hops in our setup — platform RPC (often tunneled to local 9000) vs C++ game server (often ARCADE_GAME_HOST / ARCADE_GAME_PORT, e.g. 18080). It implemented configurable platform endpoint in arcade_project/client/arcade_client.py: SERVER_HOST/SERVER_PORT from ARCADE_PLATFORM_HOST and ARCADE_PLATFORM_PORT with safe int parsing and fallback to 127.0.0.1:9000.

What I actually used: I kept the env-based platform connection patch and reused the explanation on slides/notecards. Teammates can set powershell $env:... or MobaXterm env before python client.py without editing source.
